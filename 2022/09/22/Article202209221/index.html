重装Anaconda的步骤与问题

1.卸载Anaconda

1.首先删掉anaconda根路径下envs和pkgs两个文件夹。

2.然后使用根路径下Uninstall-Anaconda3.exe来卸载anaconda。

3.将环境变量中有关anaconda的变量删除。

4.删除anaconda的注册表：我的做法是找到C:\Users\用户名\下有关anaconda的文件，这样才能彻底删除所有的anaconda的内容。

备选方案：

使用辅助工具（需要网络）

conda install anaconda-clean
anaconda-clean --yes
anaconda-clean #第一个命令是一个一个删，途中需要输y，第二个命令是不询问，直接删。

2.jupyter出现闪退情况

这个问题往往是jupyter的配置文件造成的影响。

使用以下命令输出配置文件

jupyter notebook --generate-config

然后更改jupyter的工作路径，具体属性为c.NotebookApp.notebook_dir，如下所示：

最后设置快捷方式，找到Jupyter的快捷方式，右键打开属性，将快捷方式中“目标”中的%USERPROFILE%去掉。
并且将起始位置设置为你的工作目录，并且确认即可。

3.更新anaconda

conda update conda
conda update --all

4.相关的虚拟环境命令

血的教训，在配置相关的环境之前，最好先创建一个虚拟环境。不然环境出现了库版本冲突，很容易就导致环境崩掉，只有一个base环境的很有可能就需要重新安装anaconda。

# 查看当前有哪些虚拟环境
conda env list
conda info -e
# 创建python版本为x.x并且名称为your_env_name的虚拟环境
conda create -n your_env_name python=x.x
# 激活或者切换虚拟环境
# 最新版的前面必须加conda
conda activate your_env_name
# 对虚拟环境中安装额外的包
conda install -n your_env_name [package]
# 关闭虚拟环境(即从当前环境退出返回使用PATH环境中的默认python版本)
# 最新版的前面必须加conda
conda deactivate env_name
# 删除虚拟环境
conda remove -n your_env_name --all
# 删除环境中的某个包
conda remove --name your_env_name  package_name
